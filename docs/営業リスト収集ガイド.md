# Weldex 営業リスト収集ガイド
## メールは送らず、見込み客リストだけ集める方法

> **DB は PostgreSQL（Railway）です。SQLite は廃止しました。**
> 通常の運用は本番ダッシュボード（Web）で完結します。
> 本番 URL：https://weldex-production.up.railway.app

---

## このシステムでできること

Google マップ（Places API）からターゲット企業を自動で集め、サイトの状態を診断してリスト化する。

- **集める**：業種 × エリアで企業名・電話・サイトURLを DB（PostgreSQL）保存
- **診断する**：サイトなし / 古い / スマホ未対応 / LINE未連携 などを自動判定
- **見る**：ダッシュボードでリストを確認・フィルター・電話発信
- **メールは送らない**（Phase1 はリスト収集のみ）

---

## 通常の使い方（本番ダッシュボード・推奨）

ローカルでのセットアップは不要。ブラウザだけで完結する。

1. ブラウザで **https://weldex-production.up.railway.app** を開く
2. Basic 認証（ユーザー名：`weldex` / パスワード：`DASHBOARD_PASSWORD`）でログイン
3. 業種チェックボックス → 都道府県（左）→ 市区（右）を選ぶ
4. **「収集して診断する」ボタン**を押す
   → 収集後に自動で診断まで走る（20件バッチ）
5. ターゲットリストで `none` / `old` などに絞り込み、`tel:` リンクから電話営業

> DB は Railway PostgreSQL に保存されるので、PC を閉じてもデータは消えない。

### 診断ラベルの意味

| ラベル | 意味 | 営業チャンス |
|--------|------|------------|
| `none` | サイトなし・404 | ★★★ サイト制作の提案 |
| `old` | 5年以上更新なし・SSL未対応 | ★★★ リニューアル提案 |
| `no_mobile` | スマホ非対応 | ★★ スマホ対応提案 |
| `phone_only` | 電話予約のみ | ★★ LINE予約提案 |
| `ok` | 問題なし | — 今はスルー |

## ローカル CLI（上級者向け・任意）

ローカルから直接コマンドで動かすこともできる。
**その場合も DB は PostgreSQL なので `DATABASE_URL` の設定が必須**（SQLite には保存されない）。

### セットアップ

```bash
cd ~/Desktop/weldex/outreach

# ライブラリをインストール
pip3 install -r requirements.txt

# DATABASE_URL を設定（Railway の Postgres 接続文字列）
#   Railway → Postgres サービス → Connect → Connection String をコピー
export DATABASE_URL="postgresql://...:...@...railway.app:5432/railway"

# テーブルを初期化（存在しない場合のみ作成。既存データは消えない）
python3 -c "from db.database import init_db; init_db(); print('DB OK')"
```

> `.env` の `GOOGLE_PLACES_API_KEY` は設定済みなので、API キーの作業は不要。
> `DATABASE_URL` が未設定だと「DATABASE_URL が設定されていません」エラーになる。

### ① リストを収集する

```bash
# 業種とエリアを指定して収集（1回20件ずつ）
python3 main.py collect --industry 歯科医院 --area 千葉市中央区 --limit 20
```

**業種の選択肢：**
```
歯科医院 / クリニック / 美容外科 / 整骨院
司法書士 / 税理士 / 行政書士
建設会社 / リフォーム / 美容室
```

**エリアの例：**
```
千葉市中央区 / 船橋市 / 松戸市 / 柏市
さいたま市大宮区 / 川越市 / 熊谷市
水戸市 / 宇都宮市 / 前橋市
青森市 / 秋田市 / 盛岡市
金沢市 / 富山市 / 福井市
```

### ② サイトを診断する

```bash
python3 main.py diagnose
```

各企業のサイトを自動で確認し、上記の診断ラベルを付ける。

### ③ まとめて全業種を収集する（週1回向け）

```bash
# 全業種 × 全エリアから自動収集（業種ごと5件）
python3 main.py batch-collect --limit-per-industry 5
```

> 多くとりすぎると Google API の無料枠を消費するので注意。

### ④ 条件で絞り込み表示する

```bash
python3 main.py filter --hp none,old --no-line --limit 100
```

## よくある質問

**Q. 収集した企業に迷惑はかからない？**
A. Google マップに載っている公開情報を取得するだけ。メールも送信しない。

**Q. 同じ企業が重複して入らない？**
A. `place_id`（Google固有ID）で重複排除しているので入らない（`targets.place_id` が UNIQUE）。

**Q. 「DATABASE_URL が設定されていません」と出た**
A. ローカル CLI で実行する場合は `export DATABASE_URL=...`（Railway Postgres の接続文字列）が必要。通常はダッシュボードを使えば設定不要。

**Q. データはどこに保存される？**
A. Railway の PostgreSQL（`targets` テーブル）。ローカルファイル（SQLite）には保存されない。

## データの場所

```
Railway プロジェクト
└── Postgres サービス
    └── targets テーブル  ← 収集した企業データがここに入る
```

中身を直接見たいときは Railway のダッシュボード（Postgres → Data）や、
`psql "$DATABASE_URL"` で接続して確認する。
